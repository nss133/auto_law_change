## 2. 소스별 상세

### 2.1 금융위원회(FSC)

- **목록 URL**: `https://www.fsc.go.kr/po040301`
- **카테고리**: legislation (입법예고·규정변경예고 통합)
- **HTML 파싱**:
  - 링크 선택자: `a[href*="/po040301/view?"]`
  - 상세 URL 패턴: `https://www.fsc.go.kr/po040301/view?...` 또는 `noticeId=`
  - 키 추출: path 숫자 ID 또는 `noticeId` 쿼리파라미터
  - 날짜: 부모 요소 텍스트에서 YYYY-MM-DD 패턴 추출
- **출처**: `briefing/sources/fsc.py` → `_fetch_legislation_list()`

### 2.2 금융정보분석원(KoFIU)

- **목록 URL**: `https://www.kofiu.go.kr/kor/law/legislation_list.do`
- **카테고리**: legislation (입법/규정변경 예고)
- **HTML 파싱**:
  - 테이블 행: `table tr`
  - 링크 선택자: `a[href*="view.do"]`
  - 키: `seq` 또는 `nttNo` 쿼리파라미터
  - 날짜: 행 텍스트에서 YYYY-MM-DD 패턴
- **출처**: `briefing/sources/kofiu.py` → `_fetch_list(KOFIU_LAW_NOTICE_LIST, ...)`

### 2.3 고용노동부(MOEL)

- **목록 URL**: `https://www.moel.go.kr/info/lawinfo/lawmaking/list.do`
- **카테고리**: legislation (입법·행정예고)
- **HTML 파싱**:
  - 링크 선택자: `a[href*="/info/lawinfo/lawmaking/view.do"]`
  - 키: `bbs_seq`
  - 날짜: 행 텍스트에서 YYYY-MM-DD 패턴
- **출처**: `briefing/sources/moel.py` → `_fetch_lawmaking()`

### 2.4 공정거래위원회(KFTC)

- **목록 URL**: `https://www.ftc.go.kr/www/selectBbsNttList.do`
- **파라미터**: `bordCd=105`, `key=193` (입법·행정예고 게시판)
- **카테고리**: legislation
- **HTML 파싱**:
  - 링크 선택자: `a[href*="selectBbsNttView.do"]`
  - 키: `nttSn`
  - 첨부: `downloadBbsFile`, `previewBbsAtchmnfl.do`
  - 날짜: 행 텍스트에서 YYYY-MM-DD 패턴
- **출처**: `briefing/sources/kftc.py` → `_fetch_list(KFTC_LAW_NOTICE_LIST, ...)`

## 3. 공공 API (법제처·국민참여입법센터)

| 제공처 | API | 용도 |
|--------|-----|------|
| **공공데이터포털** | 법제처 입법예고 상세 조회 | 제목, 고시번호, 예고일자, 소관부처, 예고기간, 관련법령, 제정·개정사유, 주요내용 요지 등 |
| **국민참여입법센터** | 입법예고 목록 조회 | 공포일자/수정일 기준 목록 검색 |
| **국가법령정보센터(open.law.go.kr)** | lawSearch.do?target=lsHstInf | 법령 변경이력 조회 (입법예고와는 별개) |

- **비고**: 입법예고/규정변경예고 전용 law.go.kr DRF API는 확인 필요. 현재 law_change_auto는 `lsStmd`, `admrul`(법령·행정규칙)만 사용 중.

## 4. FetchedItem 구조 (news clipping)

```python
@dataclass
class FetchedItem:
    source: SourceCode       # fsc, kofiu, moel, kftc 등
    category: Category       # "legislation"
    source_item_key: str     # 고유 키 (noticeId, bbs_seq, nttSn 등)
    title: str
    url: str
    published_at: Optional[datetime]
    attachments: list[Attachment]
    raw_text: Optional[str]  # 본문 추출 시
```

## 5. law_change_auto 연동 시 고려사항

1. **모니터링 대상 매칭**: `monitored_laws.xlsx`의 법령명·규칙명과 입법예고/규정변경예고 제목·본문 매칭
2. **데이터 소스 선택**:
   - **옵션 A**: news clipping과 동일한 HTML 목록 크롤링 (FSC, KoFIU, MOEL, KFTC)
   - **옵션 B**: 공공데이터포털 입법예고 API 활용 (정형 데이터, 추가 인증 필요)
3. **안내서 구조**: 입법예고/규정변경예고는 개정이유·주요내용·신구조문 대비표와 구조가 달라 별도 섹션/템플릿 필요
4. **중복 방지**: news clipping과 law_change_auto 간 중복 수집·발송 정책 결정 필요 (동일 URL 기준 등)
